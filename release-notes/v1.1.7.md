# Kairos v1.1.7

## Highlights

- Hardened the desktop-extension bridge authentication path and removed token transport via URL query params.
- Added owner-only filesystem permissions for bridge discovery and local telemetry/outbox databases.
- Improved extension replay resilience so malformed desktop replay results no longer stall the entire outbox queue.

## Security

- Desktop bridge now expects auth via WebSocket subprotocol token instead of `?token=` URL parameters.
- WebSocket origin checks are now restricted to extension-host/loopback-safe origins.
- Bridge discovery directory/file are now created with private permissions (`0700`/`0600`).
- Desktop SQLite and VS Code outbox SQLite files now enforce private file modes (`0600`) with private parent dirs.
- Extension discovery parsing now rejects non-loopback discovered hosts.

## Reliability

- Replay now falls back to single-event isolation for invalid desktop per-event result payloads.
- Repeatedly failing rows from invalid replay results are quarantined after threshold attempts, allowing queue progress.
- Added tests for auth subprotocol handling, origin validation, loopback discovery constraints, and permission hardening.

## Website

- Updated website and docs copy to match shipped extension UX: status bar now shows coding time only (with icon), while hover shows current session details.
- Removed outdated website references to buffering/connection labels in the VS Code status bar.
- Added website messaging to reflect bridge security hardening and resilient replay/quarantine behavior from this release.
