# Kairos v1.1.4

## Highlights

- Migrated desktop-extension transport to persistent WebSocket communication.
- Upgraded extension connectivity and health status to realtime-first behavior.
- Improved frontend freshness with push-first updates, event revision gating, and scoped refresh routing.
- Added websocket keepalive and idle reconnect handling to reduce stale connected states.

## Realtime and Reliability

- Extension now uses `/v1/extension/ws` for handshake and ingestion requests.
- Desktop server enforces protocol version checks and handshake-before-ingest sequencing.
- Server-side ping/pong keepalive marks disconnected status quickly when the bridge is stale.
- Extension runtime adds proactive connection probes and retry recovery while idle.

## Frontend Freshness

- Desktop change events now include monotonic `revision` metadata for ordering safety.
- Frontend coalesces refresh requests and rejects stale responses that lose race against newer revisions.
- Event kinds (`activity`, `sessions`, `extension-status`, `settings`) now refresh only relevant screens to reduce UI churn and latency.
- Polling remains as a low-frequency recovery fallback.

## Developer Experience

- Added a dev-only live revision badge in the desktop navbar showing latest event revision, kind, timestamp, and lag.
- Lag indicator now color-codes thresholds for quick realtime diagnostics.
