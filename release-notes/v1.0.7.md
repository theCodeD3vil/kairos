# Kairos v1.0.7

## Highlights

- Added a polished in-app "What's New" flow driven by release notes content.
- Update notifications can now open changelogs directly in app UI.
- GitHub releases now publish curated markdown release notes from the repository (not auto commit history).
- Improved desktop/extension bridge resilience with automatic local port fallback and endpoint discovery.
- Linux desktop packaging got stronger defaults for startup and icon integration.

## What's New Experience

- Update notifications now route users to changelog view automatically.
- Settings supports a changelog deep-link flow to open release notes modal directly.
- Changelog modal is streamlined for readability with markdown-rendered notes.

## Release Quality

- Desktop and extension release workflows now fail fast if curated notes are missing.
- Release bodies are sourced from `release-notes/vX.Y.Z.md` for consistent product storytelling.

## Platform Reliability

- Desktop app no longer hard-fails when the preferred local server port is already in use.
- Desktop writes active bridge endpoint metadata for extension discovery.
- Extension client now discovers and reconnects across local candidate ports automatically.
- Linux `.deb` packaging improvements include desktop entry/icon handling and runtime dependency metadata.
