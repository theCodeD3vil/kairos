# Kairos v1.1.8

## Highlights

- Added a dedicated Menubar settings tab with preset-based configuration for faster setup.
- Added Menubar presets (`Full`, `Minimal`, `Off`, `None`) with backend-enforced behavior mapping.
- Added menu bar login launch mode so Kairos can start as menu-bar-only after system login.
- Added first-launch-after-update changelog modal (shown once per installed version).

## Desktop App

- App Behavior now focuses on startup and restore preferences, while Menubar controls are centralized in their own tab.
- Menubar can now be enabled/disabled at runtime from settings without restarting the app.
- Menubar timeline now uses today's data in 2-hour buckets, and supports visibility toggles for timeline/session sections.
- Menubar popover now better respects active spaces/fullscreen behavior on macOS.

## Data & UX Improvements

- Overview 1D trend chart now supports interval filters (`5m`, `30m`, `1h`, `2h`, `6h`) with real rebucketing.
- Menu bar trendline rendering switched to a smooth sparkline curve.
- Fixed sync health history bucketing so the newest block no longer appears incorrectly degraded/offline.

## Quality

- Added and updated frontend/backend tests for menubar settings, startup/login behavior, sync health bucketing, and 1D interval trend aggregation.
- Typecheck and desktop test suites updated and passing for this release.
