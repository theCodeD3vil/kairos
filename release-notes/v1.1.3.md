# Kairos v1.1.3

## Highlights

- Hardened desktop-extension protocol contracts for durable replay safety.
- Added durable extension outbox persistence with explicit replay acknowledgements.
- Introduced extension settings mirroring with desktop-authoritative effective settings.
- Enriched desktop extension health/backlog telemetry and status persistence.
- Redesigned desktop sessionization to anchor on machine + workspace/project activity (file path optional).
- Added outbox safeguard thresholds with predictable hard-cap behavior and recovery after backlog drain.

## Reliability

- Replay now uses per-event result semantics (`accepted`, `duplicate`, `rejected_permanent`, `rejected_temporary`).
- Duplicate acknowledgements are treated as safe-to-clear without data loss risk.
- Ambiguous replay failures safely revert sending rows back to pending.
- Permanent rejects are quarantined to prevent infinite retry loops.

## Privacy and Settings

- Extension loads sane defaults on first run when desktop is unavailable.
- Cached desktop settings snapshots are used offline and replaced on handshake version changes.
- Desktop remains the canonical authority for tracking/privacy/exclusion controls.

## User Impact

- Better offline resilience with durable capture/replay behavior.
- More accurate session reconstruction for masked/hidden file-path scenarios.
- Clearer extension status reporting and outbox backlog visibility.
