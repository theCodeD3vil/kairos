# Kairos v1.0.6

## Highlights

- Desktop startup is now resilient when the local bridge port is occupied.
- Automatic local port fallback is now built in for desktop-to-extension communication.
- VS Code extension now auto-discovers the active desktop bridge endpoint.
- Linux packaging has stronger defaults for app icon and desktop integration.
- Homebrew cask distribution is now available for smoother macOS install and upgrade flows.

## Desktop

- Added fallback startup behavior for the local extension server instead of failing app launch.
- Added bridge discovery file support at `~/.kairos/desktop-bridge.json`.
- Added deterministic fallback port strategy on loopback.
- Improved Linux `.deb` metadata and icon wiring:
  - desktop entry now references installed pixmap icon
  - runtime package dependencies are declared more explicitly
  - install hooks refresh desktop/icon caches when available

## VS Code Extension

- Added runtime bridge discovery logic:
  - honors explicit `KAIROS_DESKTOP_URL`
  - reads desktop discovery file when available
  - probes localhost candidate range to reconnect automatically
- Improved connection resilience with fast local timeout handling.

## Packaging and Distribution

- Added Homebrew cask metadata and installation documentation.
- Added curated release-note publication flow for GitHub Releases.

## Notes

- This release focuses on reliability, connection recovery, and release quality.
