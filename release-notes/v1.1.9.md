# Kairos v1.1.9

## Highlights

- Added full Catppuccin-native menubar language icon resolution with alias parity to frontend mappings.
- Fixed dropdown positioning and overflow behavior so menus stay anchored and within app bounds.
- Improved analytics trend behavior for non-week ranges, including dynamic trend labeling and daily trend shaping for month/custom ranges.

## Desktop App

- Menubar popover now resolves language icon keys consistently between frontend and desktop backend.
- File/language icon handling now keeps `typescriptreact` alias behavior while preserving correct React display labels.
- Dropdown menu surfaces were refined to prevent transparent overflow artifacts and restore stable anchored placement.

## Analytics & Data

- Fixed month range trend data path so monthly trend uses daily buckets instead of weekly aggregation.
- Analytics “trend” card now adapts title and dataset by selected range:
  - `week` -> Weekly trend (weekly buckets)
  - `month` -> Monthly trend (daily buckets)
  - `today/custom` -> Today/Custom trend (daily buckets)
- Added denser reliability coverage for range and calendar boundaries, including leap-year handling.

## Security, Reliability & Performance

- Added security hardening tests for strict WebSocket JSON payload decoding and bridge token parsing/validation.
- Fixed a real websocket keepalive data race found under `go test -race` by synchronizing keepalive timing settings access.
- Added benchmark coverage for high-volume (`50k` sessions) aggregation hot paths:
  - daily totals
  - project summaries
  - language summaries
  - machine summaries
