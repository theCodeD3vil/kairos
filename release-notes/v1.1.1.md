# Kairos v1.1.1

## Fixes

- Fixed desktop update checks that could fail with `403 Forbidden` on GitHub API endpoints.
- Improved update lookup fallback for stable builds:
  - try GitHub `releases/latest` API first,
  - fall back to release list API when needed,
  - finally fall back to GitHub web release redirect parsing.

## User Impact

- Update checks are now significantly more resilient without requiring a GitHub token.
- Settings update status should no longer fail due to common unauthenticated API-rate-limit responses.
