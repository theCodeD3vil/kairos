
# Introduction
URL: /docs/introduction



<AdBanner />

## Why Joly UI?

Joly UI extends shadcn/ui with **50+ additional components** that you won't find in the original library. Same copy-paste experience, same quality, more options.

* **Zero dependencies to install** - Just copy the code into your project
* **Built on battle-tested primitives** - Uses Radix UI and shadcn/ui foundations
* **TypeScript-first** - Full type safety with strict mode support
* **Tailwind CSS v4 ready** - Modern styling with CSS variables
* **WCAG 2.1 AA compliant** - Accessible by default

## Key Features

* **Composable by design** - Build complex interfaces by combining simple, focused components
* **Extends shadcn/ui** - Same installation process, same conventions, more components
* **Copy-paste ready** - No npm packages, just copy the code you need
* **Fully customizable** - Built with Tailwind CSS and designed to match your design system
* **Advanced components** - Data tables, media players, drag-and-drop, text animations, and more

## Get Started in 2 Minutes

```bash
npx shadcn@latest add "https://jolyui.dev/r/button"
```

That's it. The component is now in your project. Customize it however you want.

## Credits

* [Radix UI](https://radix-ui.com) - Accessible component primitives
* [shadcn/ui](https://ui.shadcn.com) - Design system and registry distribution
* [Fumadocs](https://fumadocs.dev) - Documentation framework
* [Motion](https://motion.dev) - Animation library

# Troubleshooting
URL: /docs/troubleshooting



<AdBanner />

## Verify your versions

| Dependency       | Minimum recommended version |
| ---------------- | --------------------------- |
| **Motion**       | `â‰¥ 12.23.0`                 |
| **React**        | `â‰¥ 19`                      |
| **Tailwind CSS** | `â‰¥ 4.1.0`                   |
| **Shadcn UI**    | `latest`                    |

Make sure your environment meets or exceeds these versions to ensure Joly UI remains compatible and stable.

## Documentation display issues

Certain browser extensions can interfere with how the documentation is rendered. Common culprits include:

* **[Screen Shader | Smart Screen Tinting](https://chromewebstore.google.com/detail/screen-shader-smart-scree/fmlboobidmkelggdainpknloccojpppi?hl=fr\&utm_source=ext_sidebar)**

If problems persist after disabling extensions, clear your browser cache and refresh the page.

## The CLI installation does not respect aliases

In some projects, components may not install in the desired location that you configured in the aliases in the `components.json` file. This is because we can't use a target path with aliases.

We have filed an [issue on shadcn/ui](https://github.com/shadcn-ui/ui/issues/8169), Shadcn added it to the roadmap.

For now, we recommend using the manual installation.

## Flickering with the highlight component

If you are experiencing flickering issues with the `Highlight` component, this is likely due to React's strict mode. Set it to false to avoid issues during development.

## Other issues

If you encounter any other problem, don't hesitate to [open an issue here](https://github.com/johuniq/jolyui/issues/new/choose).

## Physics Options

The `physicsOptions` prop allows you to customize the physics behavior:

| Property      | Type     | Default | Description               |
| ------------- | -------- | ------- | ------------------------- |
| `restitution` | `number` | `0.8`   | Bounciness of words (0-1) |
| `frictionAir` | `number` | `0.01`  | Air resistance            |
| `friction`    | `number` | `0.2`   | Surface friction          |
| `density`     | `number` | `0.001` | Mass density of words     |

## Initial Velocity

Control the initial velocity when words start falling:

| Property  | Type     | Default | Description               |
| --------- | -------- | ------- | ------------------------- |
| `x`       | `number` | `5`     | Horizontal velocity range |
| `y`       | `number` | `0`     | Vertical velocity range   |
| `angular` | `number` | `0.05`  | Angular velocity range    |
